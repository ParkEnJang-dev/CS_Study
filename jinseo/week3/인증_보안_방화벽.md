# 🛡️ 인증, 보안, 그리고 방화벽 (Authentication & Security & Firewall)

## 1. 인증 (Authentication) vs 인가 (Authorization)

면접에서 가장 먼저 물어보는 기본 개념입니다. 두 용어를 명확히 구분해야 합니다.

* **인증 (Authentication)**: "당신은 누구인가요?"를 확인하는 과정입니다. (예: 아이디/비밀번호 입력, 생체 인식)
* **인가 (Authorization)**: "당신은 무엇을 할 권한이 있나요?"를 확인하는 과정입니다. (예: 일반 사용자는 조회만, 관리자는 삭제까지 가능)
* **주요 방식**:
* **Knowledge (지식)**: 비밀번호, PIN
* **Possession (소유)**: OTP 번호, 스마트폰 인증
* **Inherence (생체)**: 지문, 안면 인식

## 2. 웹 보안 및 암호화 (Security)

데이터를 전송하거나 저장할 때 탈취되지 않도록 보호하는 기술입니다.

### 🔹 대칭키 vs 공개키 (비대칭키)

* **대칭키 (Symmetric Key)**: 암호화와 복호화에 **같은 키**를 사용합니다. 속도가 빠르지만 키를 전달하는 과정에서 유출될 위험이 있습니다. (예: AES)
* **공개키 (Asymmetric Key)**: 암호화는 **공개키**, 복호화는 **개인키**로 합니다. 키 전달 문제가 해결되지만 연산량이 많아 느립니다. (예: RSA)
* **실제 활용**: **HTTPS(SSL/TLS)**에서는 이 두 방식을 혼합하여 사용합니다. (핸드쉐이크 때는 공개키로 대칭키를 안전하게 교환하고, 실제 데이터 전송은 빠른 대칭키로 수행)

### 🔹 해시 함수 (Hash Function)

* 어떤 데이터를 넣어도 고정된 길이의 문자열로 변환하는 단방향 함수입니다.
* **복호화가 불가능**하므로 주로 비밀번호 저장에 사용됩니다.
* **보안 강화**: 같은 비밀번호도 다른 해시값이 나오도록 임의의 값을 섞는 **Salt** 기법을 주로 사용합니다.

## 3. 방화벽 (Firewall)

네트워크 통신을 제어하여 신뢰할 수 없는 외부의 침입을 차단하는 보안 시스템입니다.

### 🔹 방화벽의 역할

* **접근 제어**: 허용된 IP나 포트만 통과시키고 나머지는 차단합니다.
* **로그 기록**: 통신 내역을 기록하여 침입 흔적을 분석합니다.

### 🔹 종류별 특징

1. **패킷 필터링 방화벽 (Packet Filtering)**: 가장 기본적인 형태로, 패킷의 헤더(IP, 포트)만 보고 통과 여부를 결정합니다. 속도는 빠르지만 패킷 내부 내용(데이터)은 확인하지 못합니다.
2. **애플리케이션 게이트웨이 (WAF, 웹 방화벽)**: 웹 요청(HTTP/HTTPS)의 내용을 분석합니다. SQL Injection이나 XSS 같은 특정 공격을 탐지하는 데 특화되어 있습니다.
3. **차세대 방화벽 (NGFW)**: 사용자가 어떤 앱을 쓰는지까지 분석하며, 침입 탐지 시스템(IDS/IPS) 기능을 통합한 형태입니다.

---

## 4. 침입 탐지(IDS) 및 방지(IPS) 시스템

방화벽만으로는 부족할 때 사용하는 보조 시스템입니다.

* **IDS (Intrusion Detection System)**: 침입을 **탐지**하고 관리자에게 알립니다. (감시 카메라 역할)
* **IPS (Intrusion Prevention System)**: 침입이 감지되면 즉시 해당 패킷을 **차단**합니다. (경비원 역할)

## 💬 면접 대비 핵심 포인트 (Q&A)

**Q. 비밀번호를 DB에 그대로 저장하면 안 되는 이유는?**

> **A:** 관리자나 해커가 DB를 열람했을 때 정보가 그대로 노출되기 때문입니다. 반드시 해시 함수와 **Salt** 기법을 사용하여 복호화가 불가능한 형태로 저장해야 합니다.

**Q. 방화벽이 있는데 왜 웹 방화벽(WAF)이 따로 필요한가요?**

> **A:** 일반 방화벽은 IP나 포트 번호(L3/L4 정보)만 보지만, 웹 방화벽은 실제 HTTP 요청 메시지(L7 정보) 내부의 악성 코드나 비정상적인 요청 패턴을 분석하기 때문입니다.

**Q. 2차 인증(2FA/MFA)이 보안에 중요한 이유는?**

> **A:** 하나의 인증 수단(지식)이 뚫리더라도, 다른 수단(소유 - 스마트폰 등)이 있어야 로그인이 가능하므로 계정 탈취 가능성을 획기적으로 낮출 수 있기 때문입니다.
