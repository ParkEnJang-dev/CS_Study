# [OS]메모리 관리 및 CPU 스케줄링 정리

## 1. 메모리 관리(Memory Management)

메모리 관리는 제한된 RAM 자원을 여러 프로세스가 효율적으로 사용할 수 있도록 관리하는 기법입니다.

### 🔹 주요 개념

* **주소 바인딩(Address Binding):**프로그램의 논리적 주소(Logical Address)를 물리적 메모리 주소(Physical Address)로 매핑하는 과정입니다.
* **스와핑(Swapping):**CPU에서 실행 중이지 않은 프로세스를 하드디스크(Swap Area)로 내보내고, 필요한 프로세스를 메모리에 올리는 작업입니다.

### 🔹 메모리 할당 방식

1. **연속 할당(Contiguous Allocation):**프로세스를 메모리의 연속적인 공간에 올리는 방식입니다. (단점: 외부 단편화 발생)
2. **불연속 할당(Non-contiguous Allocation):**현대 OS에서 주로 사용하며, 프로세스를 나누어 메모리 곳곳에 분산 배치합니다.
* **페이징(Paging):**프로세스를 동일한 크기의**페이지(Page)**로 나누어 관리합니다. (내부 단편화 발생 가능)
* **세그멘테이션(Segmentation):**의미 있는 단위(함수, 루틴 등)인**세그먼트(Segment)**로 나누어 관리합니다. (외부 단편화 발생 가능)



### 🔹 가상 메모리(Virtual Memory)

* **정의:**프로세스 전체가 메모리에 올라오지 않아도 실행이 가능하도록 하는 기법입니다.
* **요구 페이징(Demand Paging):**필요한 시점에만 페이지를 메모리에 로드합니다.
* **페이지 교체 알고리즘:**메모리가 꽉 찼을 때 어떤 페이지를 쫓아낼지 결정합니다. (FIFO, LRU, LFU 등)

## 2. CPU 스케줄링(CPU Scheduling)

CPU를 언제, 어떤 프로세스에게 할당할지 결정하는 작업입니다.

### 🔹 스케줄링 분류

* **선점형(Preemptive):**OS가 강제로 CPU를 뺏을 수 있음(현대 OS 표준). 예: Round Robin
* **비선점형(Non-preemptive):**프로세스가 스스로 CPU를 반납할 때까지 기다림. 예: FCFS

### 🔹 주요 알고리즘

1. **FCFS (First-Come, First-Served):**먼저 도착한 순서대로 처리. (단점: 콘보이 효과 발생)
2. **SJF (Shortest Job First):**실행 시간이 가장 짧은 작업 우선 처리. (단점: 기아 현상 발생)
3. **Round Robin (RR):**각 프로세스에 동일한**시간 할당량(Time Quantum)**을 주고 순서대로 처리. (가장 대중적임)
4. **Priority Scheduling:**우선순위가 높은 프로세스 우선 처리.
5. **Multi-Level Queue:**성격에 따라 큐를 여러 개 두어 차등 관리.

## 3. 기술 면접 예상 질문(Q&A)

### Q1. 외부 단편화(External Fragmentation)와 내부 단편화(Internal Fragmentation)의 차이는?

> **A:** **외부 단편화**는 총 여유 공간은 충분하지만, 공간이 조각나 있어 프로세스를 할당하지 못하는 현상입니다. **내부 단편화**는 고정된 크기의 페이지 내부에 프로세스 할당 후 남는 빈 공간이 생기는 현상을 말합니다.

### Q2. 가상 메모리와 페이지 폴트(Page Fault)에 대해 설명해주세요.

> **A:**가상 메모리는 실제 물리 메모리보다 큰 프로그램을 실행하기 위해 도입된 기법입니다. 실행 중 필요한 페이지가 메모리에 없는 상태를**페이지 폴트**라고 하며, 이때 OS는 디스크에서 해당 페이지를 찾아 메모리에 올립니다.

### Q3. 컨텍스트 스위칭 오버헤드를 줄이려면 어떻게 해야 하나요?

> **A:**CPU 스케줄링 시**시간 할당량(Time Quantum)**을 적절하게 설정해야 합니다. 너무 짧으면 컨텍스트 스위칭이 잦아져 오버헤드가 커지고, 너무 길면 응답 속도가 떨어져 FCFS처럼 작동하게 됩니다.

### Q4. 기아 현상(Starvation)과 이를 해결하는 에이징(Aging) 기법은 무엇인가요?

> **A:** **기아 현상**은 우선순위가 낮은 프로세스가 계속해서 CPU를 할당받지 못하는 상태입니다. 이를 해결하기 위해 기다린 시간이 길어질수록 우선순위를 높여주는**에이징(Aging)**기법을 사용합니다.
